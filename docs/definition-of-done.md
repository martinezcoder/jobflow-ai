# Definition of Done

A task or feature is considered **Done** only when all the following criteria are met.

## Functional

* The acceptance criteria are fully satisfied.
* The implementation solves the intended problem without introducing unnecessary complexity.

## Quality

* The code follows the project's coding conventions.
* Existing tests continue to pass.
* New tests are added when they provide value.

## Review

* The implementation has been reviewed from both a product and technical perspective.
* Any relevant documentation has been updated.

## Version Control

* At least one meaningful commit has been created following the Conventional Commits specification.
* The changes have been pushed to the remote repository.

## Project Management

* The corresponding GitHub Issue has been updated.
* The Issue has been moved to **Done** in the GitHub Project.

---

Done means ready for the next developer to pull the latest changes and continue working without additional context.
